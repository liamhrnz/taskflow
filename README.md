# TaskFlow — Transactional Task Manager

A progressive web application built incrementally across five deliverables: HTML structure, CSS design, JavaScript interactivity, REST API integration, and cloud deployment.

## Overview

TaskFlow is a personal task/ticket manager. Users can:
- ✅ Add tasks with title, description, priority, status, and due date
- ✅ View all tasks organized with color-coded badges
- ✅ Edit task details or change status (pending → in-progress → done)
- ✅ Delete completed tasks
- ✅ Persist all data to a REST API backend

## Task Data Model

| Field | Type | Values |
|-------|------|--------|
| `id` | number | Auto-generated by server |
| `title` | string | **Required** — Short task name |
| `description` | string | Optional — Additional details |
| `priority` | string | `low` · `medium` · `high` |
| `status` | string | `pending` · `in-progress` · `done` |
| `dueDate` | string | ISO date format (YYYY-MM-DD) |

## Setup Instructions

### Prerequisites
- Node.js v24+ (installed at `C:\Program Files\nodejs`)
- npm (included with Node.js)

### Quick Start

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Start the JSON Server (API backend):**
   ```bash
   npm run server
   ```
   The API will be available at: **http://localhost:3000**

3. **Open the app in your browser:**
   - File path: `taskflow/index.html`
   - Or use a local server: `python -m http.server 8000`

## REST API Endpoints

JSON Server provides a full REST API for the `/tasks` resource:

### Get all tasks
```http
GET /tasks
```
**Response (200):**
```json
[
  {
    "id": 1,
    "title": "Set up project structure",
    "description": "Create folders for css, js, and assets",
    "priority": "high",
    "status": "done",
    "dueDate": "2026-06-01"
  }
]
```

### Get a single task
```http
GET /tasks/:id
```

### Add a new task
```http
POST /tasks
Content-Type: application/json

{
  "title": "Learn REST APIs",
  "description": "Understand HTTP methods and status codes",
  "priority": "medium",
  "status": "pending",
  "dueDate": "2026-06-10"
}
```
**Response (201):** Returns the created task with auto-generated `id`.

### Update a task
```http
PUT /tasks/:id
Content-Type: application/json

{
  "title": "Learn REST APIs",
  "status": "in-progress",
  "priority": "high",
  "description": "Understand HTTP methods and status codes",
  "dueDate": "2026-06-10"
}
```
**Response (200):** Returns the updated task.

### Delete a task
```http
DELETE /tasks/:id
```
**Response (200/204):** No content or empty response.

## Usage

### In the Browser
1. **Add a task:** Fill the form and click "Add Task"
2. **Complete a task:** Click the status button to cycle through states
3. **Edit a task:** Click "Edit" to load task into form, modify, and click "Update Task"
4. **Delete a task:** Click "Delete" and confirm

### Via API (cURL or Postman)
```bash
# Add a task
curl -X POST http://localhost:3000/tasks \
  -H "Content-Type: application/json" \
  -d '{"title": "Buy groceries", "priority": "low", "status": "pending"}'

# Get all tasks
curl http://localhost:3000/tasks

# Update a task
curl -X PUT http://localhost:3000/tasks/1 \
  -H "Content-Type: application/json" \
  -d '{"title": "Buy groceries", "status": "done"}'

# Delete a task
curl -X DELETE http://localhost:3000/tasks/1
```
